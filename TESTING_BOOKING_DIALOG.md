## Тестирование в Telegram боте

### Сценарий 1: Новое бронирование
1. Отправьте `/book` боту
2. Ответьте на вопросы:
   - Имя: `Александр`
   - Телефон: `+79123456789` или `89123456789`
   - Дата: `25.03` (формат DD.MM)
   - Время: `19:30`
   - Количество гостей: `4`
3. Подтвердите: `✅ Да, подтвердить`
4. Проверьте:
   - ✅ Вы получили подтверждение в чате
   - ✅ Админы получили карточку бронирования в админском чате

### Сценарий 2: Коррекция данных
1. Отправьте `/book`
2. На вопрос имени ответьте `А` (слишком коротко)
3. Должно появиться сообщение об ошибке
4. Ответьте корректно: `Александр`
5. Продолжите диалог успешно

### Сценарий 3: Отмена бронирования
1. Отправьте `/book`
2. Заполните все данные
3. На этапе подтверждения нажмите `❌ Отмена`
4. Должно появиться сообщение "Бронирование отменено. Если хотите забронировать ещё раз, напишите /book"

### Сценарий 4: Повторное бронирование (если есть история)
1. Если у пользователя есть предыдущие брони
2. При `/book` система должна пропустить вопрос имени и телефона
3. Начать сразу с вопроса даты

## Проверка БД после бронирования

```sql
-- Проверить созданное бронирование
SELECT id, name, phone_e164, reservation_date, reservation_time, guests_count, status
FROM bookings
ORDER BY id DESC
LIMIT 1;

-- Проверить события бронирования
SELECT event_type, actor_name, created_at
FROM booking_events
ORDER BY id DESC
```

## Причины возможных ошибок

### "Некорректный формат телефона"
- ✅ Убедитесь в формате: +7XXXXXXXXXX, 8XXXXXXXXXX или 7XXXXXXXXXX
- ✅ Только цифры, +7 в начале
- ✅ Всего 10-11 цифр

### "Некорректная дата"
- ✅ Формат: DD.MM, DD.MM.YYYY или YYYY-MM-DD
- ✅ Дата не в прошлом
- ✅ Не более 365 дней в будущем

### "Некорректное время"
- ✅ Формат: ЧЧ:МММ (например: 19:30)
- ✅ Ресторан работает примерно 11:00 - 23:00

### "Диалог истек"
- Если прошло больше 1 часа между ответами
- Решение: начните заново `/book`

## Логирование для отладки

Все события логируются в таблицу `booking_events`:
- `CREATED_FROM_BOT` — бронирование создано из бота
- `ADMIN_NOTIFIED` — админ уведомлен
- `ADMIN_NOTIFY_FAILED` — ошибка при уведомлении админа
